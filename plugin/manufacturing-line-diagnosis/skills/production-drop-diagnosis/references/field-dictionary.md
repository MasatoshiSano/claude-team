# 標準概念 辞書 + ケイパビリティマップ

profile_map.py が使う「標準概念 ↔ 別名」と、「概念 → できる分析」の対応表。任意フォーマットはこの辞書で吸収する。

## 標準概念と別名(部分一致・大小無視)

| 標準概念 | 代表的な別名・表記ゆれ |
|---|---|
| 日付 | 年月日, date, 営業日 |
| 時刻 | 日時, timestamp, 生産時刻, 開始日時 |
| 計画数 | 計画, 計画台数, 目標, plan, target |
| 実績数 | 実績, 生産数, 出来高, 台数, output |
| 機種 | 品種, 型式, 品番, model, item |
| シリアル | serial, 製造番号, 個体, SN |
| 良品数 | 良品, 合格, OK |
| 不良数 | 不良, 不適合, NG, defect |
| 不良項目 | 不良内容, 不良モード, 欠陥 |
| 発生工程 | 発生 |
| 流出区分 | 流出, 検出 |
| 停止時間 | 停止, ダウン, downtime, 故障時間 |
| 停止理由 | 事象, 理由, 現象, reason |
| 停止区分 | 区分, 分類, ロス区分 |
| 設備 | 号機, machine, 設備ID, 装置, 機械 |
| ライン | line |
| 工程 | process |
| 拠点 | 工場, site, plant |
| 標準サイクル/PPH | 標準時間, サイクルタイム, CT, タクト, 標準PPH |
| 投入数 | 投入, 受入, 着工, input |
| センサ値 | 振動, 温度, 電流, 圧力, vibration |
| 保全履歴 | WO, 作業指示, 故障コード, maintenance |
| 4M変化 | 変化点, 変更内容, change |
| 作業者 | 担当, operator, 要員, 氏名 |
| 手順/基準書 | 標準書, 検査基準, SOP, 基準書 |

新しい別名に出会ったら、ユーザーに確認のうえこの表に追記して再現性を上げる。

## ケイパビリティマップ(概念 → できる分析)

| 分析 | 必要な標準概念 |
|---|---|
| 製造三角図(日次)・達成率・悪化起点 | 日付 + 計画数 + 実績数 |
| 製造三角図(日内) | 時刻 + 実績数 |
| 計画/実績PPH(機種別) | 時刻 + 機種 + 実績数 |
| 機種ズレ(遅れ型/順序) | 時刻 + 機種 + 計画数 |
| 流動数曲線・仕掛 | 日付 + 投入数 + 良品数 |
| 稼働率A・ダウンタイム時系列 | 時刻 + 停止時間 |
| ダウンタイムPareto(区分/設備) | 停止時間 + 設備 |
| 性能P・OEE | 標準サイクル + 実績数 + 停止時間 |
| 品質Q・不良Pareto | 良品数 + 不良数 |
| 発生/流出分析 | 不良数 + 発生工程 + 流出区分 |
| bad-actor(慢性設備) | 保全履歴 + 設備 |
| 予兆・状態基準保全(P-F) | センサ値 + 時刻 |
| RCA起点(4M変化点) | 4M変化 |
| 人要因(シフト相関) | 作業者 |

「今あるデータでできること」=必要概念が全て揃う分析。「あると良いデータ→できること」=不足概念が1つ埋まれば解禁される分析。
