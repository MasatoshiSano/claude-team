# 読み筋・定義リファレンス(progressive disclosure)

SKILL.md本体を補う詳細。必要時に参照する。

## OEE 定義(このテンプレート準拠)
- 稼働率 A = 実働 / (実働 + 停止)   ※停止=計画外(設備故障/チョコ停/軽微)。段取り(CO)・立上げ(SU)・初物検査(FP)は別管理。
- 性能 P = 標準時間 / 実働
- 品質 Q = 良品 / 実績
- OEE = A × P × Q
世界クラス85%+、一般的ディスクリート55–65%。制約工程の歩留まり改善は能力拡張と等価。

## 製造三角図 / 流動数曲線の読み
- 製造三角図: 横軸=時刻、縦軸=累計個数。計画線(対角)と実績線。実績>計画線で予定通り、下回りで遅れ。
- 実績線が横ばい=設備停止 / 傾き<計画=効率低下 / 傾きばらつき=リズム不良 / 立上がり遅れ=朝礼・初物検査。
- 流動数曲線: 累計投入 と 累計完成。縦差=仕掛(WIP)、横差=リードタイム。2本が開く=滞留(ボトルネック)。

## 計画・実績PPH(時間×機種)
- 機種をそろえて計画PPH vs 実績PPH でペース差を見る(機種で標準PPHが違うため単純総数比較は誤診)。
- 計画機種≠実績機種:
  - 順序保持の右シフト=遅れ型(多い)。起点機種の遅れ内訳=段取り超過/稼働PPH不足/前工程の遅れ継承。
  - 順序入替=順序間違い型。指示・ディスパッチ・方法。

## ダウンタイム分析
- 区分: 設備故障(BR)/段取り(CO)/段取り立上げ(SU)/初物検査(FP)/チョコ停(MS)/軽微停止(MN)。
- 区分別・設備別のPareto(構成比・累計%)で重点を特定。設備別が制約工程に集中していないか。
- 故障は MTBF/MTTR、段取りは SMED(内外段取り分離)。突発故障と段取り由来は分けて読む。

## 4M(人・機械・材料・方法)
- 原因を横に広げる骨組み。+測定/環境で4M1E/5M1E。
- 4M変化管理: 人(新人配置)・機械(治具交換)・材料(ロット切替)・方法(条件変更)の変化点を不良リスクとして管理。

## 発生原因 / 流出原因
- 発生原因: なぜその不良が作られたか → 発生防止(工程・条件)。
- 流出原因: なぜ検出されず次工程/客先へ流れたか → 流出防止(検査・検出強化)。
- 1不良につき必ず両方。FMEAの Occurrence と Detection に対応。

## RCAの作法(root-cause-analysis-copilot に接続)
- 問題定義→タイムライン→4M/物理人潜在で仮説→証拠で Verified/Hypothesis/Refuted→対策。
- 「人のミス」を終点にしない。複数原因を許す。樹形図は都度生成、作法のみ固定。
